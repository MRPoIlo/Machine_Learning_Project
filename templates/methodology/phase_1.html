Objectives

Translate a public-health question — how risky is rural drinking water across Colombian municipalities? — into a measurable Machine Learning target.

The official IRCArural index (Índice de Riesgo de Calidad del Agua para Consumo Humano, rural) gives a numeric reading from 0 to 100. Each value is also mapped to a five-class label, which is the variable we plan to predict.

Stakeholders

  • Local sanitation authorities deciding where to direct treatment investments.
  • The Ministry of Health and Social Protection, owner of the IRCA dataset.
  • Rural communities affected by uneven access to safe drinking water.
  • Academic and NGO teams researching water access in Colombia.

Data sources

Primary dataset. IRCA rural by municipality, published on datos.gov.co. Already cleaned and filtered to 2021–2024 by the team.

Secondary sources to integrate during Phase 2.

  • Precipitation — datos.gov.co/Precipitación, same 2021–2024 window. Values in millimetres bucketed into four intensity classes: 0–2 light, 2–10 moderate, 10–30 strong, >30 very strong.
  • Water infrastructure indicators (treatment plants, coverage by municipality).

Data pipeline — main steps

High-level outline of how raw open data becomes the modelling table. Each step is documented in versioned notebooks/scripts for reproducibility.

  1. 1. Source selection

    Pull historical climate records (precipitation, temperature, streamflow) and infrastructure inventories (rural aqueducts, wells, intakes) from IDEAM, DANE and SIVICAP. Filter to municipalities classified as rural by DANE and fix the temporal window (at least 10 years of climate data).

  2. 2. Data cleaning

    Handle missing values in climate series (stations with temporal gaps via IDW spatial interpolation). Reconcile inconsistencies in aqueduct coverage. Harmonise municipal codes across sources using DIVIPOLA. Drop duplicates, erroneous rows and values outside the statistical range.

  3. 3. Transformation and feature creation

    Compute water-deficit indices (accumulated precipitation vs. regional minimum threshold). Aggregate climate data by municipality and quarter. Cross with infrastructure coverage. Build the four-level water availability target variable. Normalise continuous numeric features.

  4. 4. Pipeline documentation

    Record every transformation: tools used (Python / pandas / GeoPandas), source file versions, cleaning criteria and decisions taken. Keep work in Jupyter Notebooks or scripts versioned in Git so the pipeline stays reproducible.
