Evaluation methodology

Each model is evaluated with the metrics that match its learning paradigm. We avoid forcing one set of metrics on every model: applying classification accuracy to K-Means, for example, would be misleading because the algorithm is unsupervised and has no ground-truth labels for its clusters.

  • Supervised models (Logistic Regression and LDA) — evaluated on the held-out test fold of the IRCA rural dataset using classification metrics: accuracy, precision, recall, F1-score, ROC curve and confusion matrix.
  • Unsupervised model (K-Means) — evaluated with intrinsic clustering metrics on the scaled feature matrix: silhouette score, Davies-Bouldin index and Calinski-Harabasz score.
  • Reproducibility — every split uses random_state=42 and stratify=y for the binary classifiers, so the numbers in this page match the ones reported in the R3A2 deliverable.

Metrics glossary

Classification metrics

Metric Formula / Idea What it tells us
Accuracy (TP + TN) / total Overall fraction of correct predictions. Misleading on imbalanced data.
Precision TP / (TP + FP) Of the rows the model flagged as positive, how many actually were.
Recall TP / (TP + FN) Of the truly positive rows, how many the model managed to catch.
F1-Score harmonic_mean(P, R) Single number that balances precision and recall.
ROC AUC Area under the TPR vs FPR curve Threshold-independent ranking quality. 0.5 = random, 1.0 = perfect.
Confusion matrix [[TN, FP], [FN, TP]] Breakdown of where the model agrees and disagrees with the truth.

Clustering metrics (used for K-Means)

Metric Range What it tells us
Silhouette [-1, 1] — higher is better How well each point fits its cluster versus the next nearest one.
Davies-Bouldin [0, ∞) — lower is better Average similarity between each cluster and its closest neighbour.
Calinski-Harabasz [0, ∞) — higher is better Between-cluster vs within-cluster dispersion ratio.

Best model selection

For the deployed prediction system we need a model that takes a single record (department + year) and returns whether the rural water in that municipality is at high risk. K-Means is unsupervised and cannot answer that question; it stays as an exploratory tool for the precipitation dataset. The decision is therefore between Logistic Regression and LDA.

★

Selected model — Logistic Regression

Pending the final LDA numbers, Logistic Regression is the current selection for deployment because:

  • Probability calibration. The model emits a well-defined probability between 0 and 1, which the prediction UI shows directly to the user.
  • Recall on the dangerous class. Catches {{ "%.0f"|format(logistic_eval.per_class[1].recall * 100) }}% of the High risk municipalities in the test fold, which is the metric that matters for a public-health decision-support tool.
  • Interpretability. Linear coefficients per department give a transparent explanation of why a prediction came out the way it did.
  • ROC AUC = {{ "%.3f"|format(logistic_eval.roc_auc) }} — clearly better than random (0.5), confirming the model is learning real geographic signal.

Once the LDA metrics are available we revisit this decision: if LDA matches or beats Logistic on recall and AUC and remains interpretable, we switch.
