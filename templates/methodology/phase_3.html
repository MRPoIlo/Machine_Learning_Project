Models overview

For Phase 3 the team trained three models covering both learning paradigms. Each model was selected to answer a different question on the same public-health domain.

M1

K-Means

Unsupervised. Groups the 33 Colombian weather stations into clusters of similar precipitation behaviour and geography.

M2

Logistic Regression

Supervised binary classifier. Predicts whether a rural municipality falls in the unsafe water class (Alto riesgo / Inviable sanitariamente).

M3

Linear Discriminant Analysis

Supervised classifier built by a teammate. Complements the logistic model with a generative, variance-aware approach.

1 · K-Means Clustering

Unsupervised model · IRCA rural dataset

K-Means is an unsupervised clustering algorithm that groups records into K clusters by minimising the squared distance between each point and its cluster centroid. We apply it to the same rural drinking-water dataset Logistic Regression uses, so both models are trained on identical rows and the results can be compared side by side.

Problem framing

Unlike Logistic Regression, K-Means has no target variable. It looks at the feature space and asks: which records naturally group together? We use it to discover profiles of municipal water quality — clusters of records that share department, year and IRCA fingerprint. Because there is no target to leak into, IRCArural is allowed as a feature here (it was excluded in Logistic for that reason). The optimal number of clusters K is chosen automatically by the silhouette score.

Feature engineering

  • One-hot encoding of DepartamentoCodigo: 29 binary columns, identical to the encoding used by Logistic Regression. The code is preferred over the department name because the same department appears with two spellings.
  • Temporal feature: Año kept as a numeric variable to capture the 2021–2024 trend.
  • Risk signal: IRCArural kept as a numeric feature. Allowed because K-Means is unsupervised — there is no target to leak into.
  • Dropped: MunicipioCodigo (508 unique values would explode the feature space) and the categorical risk label (we want clusters to be derived from raw data, not from the outcome).
  • Scaling: StandardScaler on the full feature matrix so all 31 features share a comparable scale before the distance-based clustering.

Training configuration

  • Algorithm: sklearn.cluster.KMeans with Lloyd's algorithm.
  • K search range: K = 2 … 8, scored with the silhouette coefficient.
  • Best K: {{ kmeans_eval.metrics.best_k }}, the value that maximised silhouette on the scaled feature matrix.
  • Initialisation: n_init=10 — the algorithm runs ten times with different centroids and keeps the run with the lowest inertia.
  • Random state: 42 for reproducible cluster labels across runs.
  • PCA (visualisation only): n_components=2 on the scaled matrix to plot clusters; first two components explain {% for v in kmeans_eval.metrics.pca_variance %}{{ "%.1f"|format(v * 100) }}%{% if not loop.last %} + {% endif %}{% endfor %} of the variance.

2 · Logistic Regression

Supervised model · IRCA rural dataset

Logistic Regression is a linear supervised classifier. It outputs the probability of a record belonging to the positive class through the sigmoid function applied to a linear combination of the features. We apply it to the rural drinking-water dataset to predict whether a municipality falls in the unsafe-water class.

Problem framing

The original column Nivel de riesgo rural contains five ordinal categories. For this binary classifier we collapse them into two:

  • High risk (1): Alto riesgo + Inviable sanitariamente. These are the cases where rural drinking water is actually unsafe and policy action is required.
  • No high risk (0): Sin riesgo, Bajo riesgo, Riesgo medio. Water is within acceptable bounds even if monitoring is still needed.

We deliberately exclude IRCArural from the features: the risk level is computed from that index by fixed thresholds, so including it would be target leakage and the model would only memorise the formula. The remaining features force the model to learn geographic and temporal patterns.

Feature engineering

  • One-hot encoding of DepartamentoCodigo: department codes are categorical identifiers, not continuous magnitudes, so they are expanded into 30 binary columns.
  • Temporal feature: Año kept as a numeric variable to capture the 2021–2024 trend.
  • Dropped: MunicipioCodigo (508 unique values would explode the feature space without generalising) and IRCArural (target leakage as explained above).
  • Scaling: StandardScaler on the assembled matrix so all features share the same scale.

Training configuration

  • Algorithm: sklearn.linear_model.LogisticRegression
  • Split: train_test_split with test_size=0.2 and stratify=y
  • Random state: 42 for reproducibility
  • Solver iterations: max_iter=1000 to guarantee convergence
  • Class weights: class_weight="balanced" — re-weights the loss so the minority class (high risk) is not ignored. This is the class we actually want to detect.

3 · Linear Discriminant Analysis

Supervised model · IRCA rural dataset

Linear Discriminant Analysis is a generative supervised classifier. It models the distribution of each class as a Gaussian with shared covariance and assigns each point to the class with the highest posterior probability. It complements Logistic Regression with a variance-aware view of the same data.
